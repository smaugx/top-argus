系统情况

关于

系统命名

系统取名为 TOP-ARGUS(TOPARGUS)，中文名为 TOP 全网监控系统。其中 ARGUS 一词来源于希腊神话，是百眼巨人阿耳戈斯·潘诺普忒斯的名字。

阿尔戈斯Argus是希腊神话中的百眼巨人。他有100只眼睛，分布在全身，口语观察到各个方向发生的事情，即使睡觉时也有两只眼睛睁着。用来比喻“警惕的人”。

本系统取名为 ARGUS，寓意能实时监控全网节点状态。包括虚拟节点总数，物理节点总数，节点奔溃，实时收发包情况，全网丢包率，逻辑网络分布，逻辑网络中节点列表，全网 CPU 分布图，全网流量分布图等。

本系统定位为网络层面的监控系统，后续希望能单独形成一个项目，主要配合开发自测、QA 测试、自动化测试、版本升级测试、主网监控等。

说明

  • 下述各个指标，没有特别说明的情况下均具有实时性，能借此实时监控主网情况（比如节点掉线，网络波动等）
  • 测试版采样率为 100%，即每一条消息的收发包情况均采集；实际随着压力增大可以适当降低采样率
  • 目前重点关注广播收发包情况，单播采样选项暂时关闭
  • 由于开发时间不足以及前端技能小白，还有很多采集到的项没有进行前端展示。同时欢迎前端同学对前端开发涉及到的框架、技术给建议

首页

首页 展示了全网总览状态，分为 3 个图表。

  • 主网虚拟节点分布：统计了全网各个逻辑网络的节点占比，全网逻辑网络分布。点击饼图的某一片，会链接到该逻辑 P2P 网络的展示页面，下面会说到
  • 主网物理节点分布： 统计了全网总的物理节点总数以及 IP 分布
  • 全网丢包率走势图： 反应了目前全网整体的丢包率情况，根据此图的波动，可以迅速感知到全网的网络状态，从而快速定位问题

其中全网丢包率的计算公式为：

全网丢包率 = SUM(每一条消息的丢包率） / 统计总数

收包情况

收包情况 展示了某个 P2P 网络的收包情况，测试版采样率 100%，实际压测的时候可以适当调整采样率。同时目前关闭了单播采集选项，只关注广播收发包情况。

页面上展示了每一条消息的收发情况，主要包含下述几个字段：

chain_hash

消息唯一 hash，点击该项可以详细查看这条消息的收包情况

chain_type

链消息类型，具体可以参考相关代码

msg_size

链消息的大小（字节），表示从 vhost 层传到网络层的 binary data 大小

packet_size

经过网络层封装过后的 packet 大小（字节），主要多加了 protobuf 结构体， xbase header

recv_nodes

该条消息最终收到的节点总数，这里的收到只过滤之后的收到，即去重之后的收到

dest_network

目标网络的节点总数，通常也就是理论上该消息最终需要被多少的节点收到，100 减去 recvnodes 与 destnetwork 的比值即为丢包率。点击该项可以链接到页面 P2P网络

drop_rate

表示该条消息的丢包率。100 减去 recv_nodes 与 dest_network 的比值，单位 %。丢包率超过一定数值（比如 10) 会显示为红色

hop_num

汇总的收包跳数，即最终每个节点收到包显示的包的跳数，根据跳数的分布，占比以及最大值可以衡量网络状况、性能、以及是否存储逻辑 BUG。

会显示为4或者5个数字，以逗号分隔(左闭右开区间）。每一项分别表示 0-5 跳的总个数； 5-10 跳的总个数；10-15 跳的总个数；15-20 跳的总个数； 20 跳以上的总个数

taking

汇总后收包的传播时间分布，即最终每个节点收到包后的传播时间，根据传播时间的分布，占比以及最大值可以衡量网络状况、性能、以及是否存储逻辑 BUG。

同 hop_num 一样，会显示为 4 或者 5 个数字，以逗号分隔（左闭右开区间）。每一项分别表示 0-500 ms 的总个数； 500 - 1000 ms 的总个数；1000-1500 ms 的总个数； 1500-2000 ms 的总个数； 2000 ms 以上的总个数；

taking 字段能比较准确和灵敏的反应网络整体状态，建议多关注下该值!

src_node_id

消息的发起者。

点击该字段会链接到类似于 当前页面 的页面，会具体显示该网络的收包情况。

dest_node_id

消息的接收者（一般表示该节点对应的网络，因为目前重点关注广播情况，关闭单播采集选项）。

点击该字段会链接到类似于 当前页面 的页面，会具体显示该网络的收包情况。

timestamp

表示该条消息存储到 db 的时间戳，与消息的实际收发时间可能会有一定的偏差（<= 5 min)。

默认以 timestamp 降序排序，即页面显示最新的收发包情况。

P2P网络

P2P网络 页面展示某个具体网络的节点列表以及该网络的整体丢包率走势图；

默认显示 Bootstrap 网络的节点列表以及全网丢包率。从 首页 的 主网虚拟节点分布 饼图的每一片跳转过来可以显示某个逻辑 P2P 网络的节点列表以及该网络的丢包率。

这里需要注意的是，默认会显示全网的丢包率，即不区分逻辑 P2P 网络以及物理网络；从饼图跳转过来会显示某个具体逻辑 P2P 网络的整体丢包率。

系统情况

系统情况 展示网络整体的系统资源消耗占比，比如整体的 CPU 分布图、整体的流量分布图。
